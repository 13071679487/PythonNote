*TCP_Server.py*

```python
import socket
import threading as th

HOST = '192.168.1.101'
PORT = 10024
BUFFER = 1024
clientList = []
serverSock =None

class MyThread(th.Thread):
    def __init__(self, socket, addr):
        super(MyThread, self).__init__()
        self.socket = socket
        self.addr=addr

    def run(self):
        print(f'新客户端连接=={self.addr}')
        while True:
                try:
                    msg = self.socket.recv(BUFFER)
                    if msg:
                        print('接收到消息:{}'.format(msg.decode('utf-8')))
                    else:
                        clientList.remove(self.socket)
                        self.socket.close()
                        print('下线了:{}'.format(self.addr))
                except:
                    pass

def beginListen():
    global serverSock , clientList
    while True:
        try:
            newSocket, addr = serverSock.accept()
            t = MyThread(newSocket, addr)
            t.start()
            clientList.append(newSocket)
        except:
            pass

if __name__ == '__main__':
    serverSock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    serverSock.setblocking(False)
    serverSock.bind((HOST, PORT))

    serverSock.listen(5)
    print('服务器已启动')
    th.Thread(target=beginListen).start()
    while True:
        msg = input('请回复\n')
        if msg:
            for client in clientList:
                client.send(msg.encode('utf-8'))
```

```python
2、进程与线程的区别：
    1、进程：
        1、进程创建与开销较大
        2、进程之间的数据不能共享，如果要实现数据的共享只能通过进程间的通信，一个进程可以有多个线程
        3、如果是有多个不同的任务，则需要使用不同的进程去处理
        4、进程之间的空间是独立，数据的安全性比较高

    2、线程：
        1、进程创建与开销较小
        2、线程之间的数据是可以共享
        3、不同通过多个线程去拼凑成一个进程
        4、线程之间的共享数据，所有数据的安全性比较低

3、多进程与多线程的区别：
    1、多进程：
        稳定性高，一个子进程的奔溃不会影响其他进程的运行
        创建代价比较高，对Cpu的要求比较高，如果任务太多，会影响操作系统无法正常调度

    2、多线程：
        稳定性比较差，一个线程的奔溃会影响整个进程
        多线程的执行效率比多进程效率高（在相同执行环境下）
```
